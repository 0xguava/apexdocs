## Variables (`var`, `let`, `const`) and Scope
In JavaScript, variables are used as symbolic names for values. The modern standard (ES6+) dictates using `const` by default for all variables. Use `let` only when 
you know the variable's value must be reassigned later. Avoid `var` entirely to prevent scope leakage and unexpected hoisting behavior.

- **`var`**: Declares a variable, initializing it to `undefined` via hoisting in the function or global scope.
- **`let`**: Declares a block-scoped variable that cannot be accessed before its initialization (Temporal Dead Zone).
- **`const`**: Declares a block-scoped variable with an immutable memory binding (the identifier cannot be reassigned).

## Datatypes, Operators, Conditionals, and Loops
JavaScript is dynamically typed and uses various structures for logic and iteration. Control flow structures include `if...else` statements and the `switch` statement 
for multi-branch logic based on strict equality. The ternary operator (`condition ? trueExpr : falseExpr`) provides a concise conditional expression.

### Data Types:
- **`String`**: Primitive representing textual data.
- **`Number`**: Primitive representing both integers and floating-point numbers.
- **`BigInt`**: Primitive representing integers with arbitrary precision.
- **`Boolean`**: Primitive representing a logical entity (`true` or `false`).
- **`Undefined`**: Primitive indicating a declared variable lacks an assigned value.
- **`Symbol`**: Primitive representing a unique, immutable identifier.
- **`Null`**: Primitive representing the intentional absence of an object value.
- **`Object`**: Complex structure representing a collection of keyed properties.

### Core Operators:
- **`==`** (Loose Equality): Compares two values for equality after converting both to a common type.
- **`===`** (Strict Equality): Compares two values for equality without type coercion; returns false if types differ.
- **`&&`** (Logical AND): Returns true if both operands are truthy.
- **`||`** (Logical OR): Returns true if at least one operand is truthy.
- **`!`** (Logical NOT): Reverts the boolean value of the operand.
- **`??`** (Nullish Coalescing): Returns the right-hand operand when the left is `null` or `undefined`.
- **`**`** (Exponentiation): Raises the first operand to the power of the second.

### Loops & Iteration:
- **`for`**: Executes a block of code a specified number of times using an initialization, condition, and increment.
- **`while`**: Executes a block of code as long as a specified condition evaluates to true.
- **`do...while`**: Executes a block of code once before checking the condition, then repeats as long as it's true.
- **`for...in`**: Iterates over the enumerable string properties (keys) of an object.
- **`for...of`**: Iterates over the iterable values of objects like Arrays, Strings, or Maps.

## Functions: Parameters vs. Arguments, Scope, and Types
Functions are first-class objects in JavaScript. Parameters are the named variables declared in the function definition (the placeholders). Arguments are the actual values 
passed to the function when it is invoked. Functions create their own local execution context (lexical scoping), meaning variables declared inside are not accessible outside.

- **`function`**: Keyword used to declare or express a reusable block of code.
- **`() => {}`** (Arrow Function): A shorter syntax for a function expression that lexically binds its own `this` context (does not have its own `this`).
- **IIFE** (Immediately Invoked Function Expression): A function that runs as soon as it is defined.

## Arrays: Properties and Methods
Arrays are global objects used to store multiple items under a single variable name. They are resizable and can contain a mix of different datatypes.

- **`length`** (Property): Returns the integer number of elements contained in the array.
- **`push()`**: Adds one or more elements to the end of an array.
- **`pop()`**: Removes the last element from an array.
- **`shift()`**: Removes the first element from an array.
- **`unshift()`**: Adds one or more elements to the front of an array.
- **`splice()`**: Modifies an array by removing, replacing, or adding elements in place.
- **`map()`**: Returns a new array populated with the results of calling a provided callback on every element.
- **`filter()`**: Returns a new array containing only elements that pass the condition implemented by the callback.
- **`reduce()`**: Executes a reducer callback on each element, resulting in a single accumulated output value.
- **`forEach()`**: Executes a provided callback function once for each array element.

## Objects: Features and Methods
Objects are collections of key-value pairs, where keys are strings (or Symbols) and values can be any datatype. Object properties can be accessed via dot notation (`obj.key`) 
or bracket notation (`obj["key"]`). Inside an object method, the `this` keyword refers to the object itself.

- **`Object.keys()`**: Returns an array of a given object's own enumerable string-keyed property names.
- **`Object.values()`**: Returns an array of a given object's own enumerable string-keyed property values.
- **`Object.entries()`**: Returns an array of a given object's own enumerable string-keyed property `[key, value]` pairs.
- **`Object.assign()`**: Copies all enumerable own properties from one or more source objects to a target object.

## DOM: Manipulation Methods

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content.

- **`document.getElementById()`**: Returns the element whose `id` property matches the passed string.
- **`document.querySelector()`**: Returns the first element that matches the specified CSS selector.
- **`document.querySelectorAll()`**: Returns a static `NodeList` of all elements matching the specified CSS selector.
- **`element.textContent`**: Property that gets or sets the text content of a node and its descendants.
- **`element.innerHTML`**: Property that gets or sets the HTML markup contained within the element.
- **`element.setAttribute()`**: Sets the value of an attribute on the specified element.
- **`element.classList.add()`**: Adds one or more CSS classes to an element's class list.
- **`element.classList.remove()`**: Removes one or more CSS classes from an element's class list.
- **`document.createElement()`**: Creates a new HTML element specified by the tag name.
- **`element.appendChild()`**: Appends a node as the last child of a parent element.
- **`element.append()`**: Inserts a set of Node objects or strings after the last child of the element.

## Event Handling: Types and Mechanisms
Events are actions or occurrences that happen in the system being programmed, which the system tells you about so your code can react.

- **`EventTarget.addEventListener()`**: Registers a specific event handler callback on a specific target (like a DOM node). Allows multiple handlers to be attached to a single event.
- **`e.target`**: An event object property that identifies the DOM element that triggered the event.
- **`e.preventDefault()`**: An event object method that cancels the browser's default behavior for that event (e.g., stopping a form submission).

## Event Loop

JavaScript is single-threaded, meaning it can only execute one piece of code at a time. The -*Event Loop** is a concurrency model responsible for executing the code,
collecting and processing events, and executing queued sub-tasks. It continuously checks if the Call Stack is empty; if it is, it pushes the oldest microtask, 
then macrotask, onto the stack for execution.

## Callbacks: Why?
A callback is a function passed into another function as an argument, which is then invoked inside the outer function to complete a routine. Historically, callbacks were 
the primary way to handle asynchronous operations. Without callbacks, the main thread would block while waiting for a network request to finish. They ensure a 
function runs right after a task has completed.

## Promises and Sync vs. Async (`async`/`await`)
Promises were introduced to solve "Callback Hell" and inversion of control, providing a cleaner, chainable syntax for handling asynchronous operations. Synchronous code 
executes sequentially and blocks further execution, while asynchronous code executes in the background.

- **`Promise`**: An object representing the eventual completion (or failure) of an asynchronous operation and its resulting value.
- **`.then()`**: Attaches callbacks for the resolution of a Promise.
- **`.catch()`**: Attaches a callback for the rejection of a Promise.
- **`.finally()`**: Attaches a callback that runs regardless of whether the Promise resolved or rejected.
- **`async`**: Keyword placed before a function to ensure it always returns a Promise.
- **`await`**: Keyword used inside an `async` function to pause execution until a Promise settles, allowing asynchronous code to be written in a readable, top-down synchronous style.
